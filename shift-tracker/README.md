# Учёт смен сотрудников

Веб-приложение (PWA) для учёта рабочих смен. Устанавливается на планшет, сотрудники
входят по ПИН-коду и отмечают начало/конец смены. Все данные хранятся на сервере
(в облаке), поэтому вы как администратор можете просматривать и редактировать все
смены как с этого же планшета (по своему ПИН-коду), так и с любого другого
устройства — телефона или компьютера, открыв ту же ссылку в браузере.

## Как это работает

**Сотрудник** (планшет, `/index.html`):
1. Вводит свой ПИН-код (4–6 цифр).
2. Если смена не открыта — нажимает «Начать смену».
3. Если смена открыта — нажимает «Завершить смену» и указывает, сколько минут длился перерыв.
   Отработанное время считается автоматически: (конец − начало) − перерыв.
4. Может посмотреть «Мои смены» — только свою историю. Экран автоматически
   возвращается к вводу ПИН-кода через 45 секунд бездействия, чтобы следующий
   сотрудник не увидел чужие данные.
5. После завершения смены сотрудник ничего в ней изменить не может — редактирование
   доступно только администратору.

**Администратор** (`/admin.html`, отдельная ссылка внизу экрана киоска):
- Вход по отдельному ПИН-коду администратора.
- Видит и редактирует смены всех сотрудников, может добавить смену вручную,
  удалить ошибочную запись.
- Управляет списком сотрудников: добавляет, меняет ПИН-код, отключает уволенных
  (их ПИН перестаёт работать, но история сохраняется).
- Экспортирует отфильтрованный список смен в Excel (.xlsx) одной кнопкой.
- Тот же admin.html открывается с любого устройства — токен входа хранится в
  браузере 12 часов.

## Локальный запуск (проверить перед деплоем)

Требуется Node.js 18+.

```bash
cd shift-tracker
npm install
cp .env.example .env      # и поменяйте ADMIN_PIN / JWT_SECRET
npm start
```

Откройте `http://localhost:3000` — это экран киоска, `http://localhost:3000/admin.html` —
админ-панель. Первый администратор создаётся автоматически при первом запуске
с ПИН-кодом из `ADMIN_PIN` (по умолчанию `9999` — обязательно смените после
первого входа: Админ-панель → Сотрудники → ✎ у своей записи → новый ПИН).

Данные хранятся в файле `data/shifts.json` рядом с приложением.

## Деплой в облако (рекомендуется Render.com — есть бесплатный план)

1. Загрузите папку `shift-tracker` в GitHub-репозиторий (или используйте
   Render → "Deploy from a Git repository").
2. На https://render.com создайте **Web Service**, укажите репозиторий.
   - Build Command: `npm install`
   - Start Command: `npm start`
3. **Обязательно** добавьте постоянный диск (Render → Disks → Add Disk),
   смонтируйте его в `/opt/render/project/src/data`, и задайте переменную
   окружения `DB_PATH=/opt/render/project/src/data/shifts.json`.
   Без диска данные будут стираться при каждом перезапуске сервиса.
4. В Environment добавьте переменные:
   - `JWT_SECRET` — любая длинная случайная строка
   - `ADMIN_NAME` — ваше имя
   - `ADMIN_PIN` — ПИН администратора при первом запуске (потом смените в панели)
5. После деплоя Render выдаст постоянную ссылку вида `https://your-app.onrender.com`.
   Откройте её на планшете — это и есть адрес киоска. На своём телефоне/компьютере
   откройте `https://your-app.onrender.com/admin.html`.

Подойдёт и любой другой хостинг с Node.js и постоянным диском (Railway, Fly.io,
свой VPS) — принцип тот же: `npm install && npm start`, плюс переменные окружения
и постоянное хранилище для `data/`.

## Установка на планшет как приложение (PWA)

1. Откройте ссылку на киоск в Chrome/Safari на планшете.
2. Меню браузера → «Добавить на главный экран» / «Установить приложение».
3. Иконка появится на рабочем столе и будет открываться в полноэкранном режиме,
   без адресной строки — как обычное приложение.

## Ограничения и что можно улучшить дальше

- ПИН-код — это код для кассового/киоскового устройства, а не полноценный
  пароль: не используйте один и тот же ПИН для входа в другие системы.
- Хранилище — JSON-файл на диске сервера, этого достаточно для 10–15 сотрудников
  и одного планшета. Если понадобится больше точек/нагрузки — можно перейти на
  полноценную БД (Postgres) без изменения интерфейса.
- Можно добавить: расчёт зарплаты по ставке, уведомления о забытой открытой смене,
  несколько точек с привязкой сотрудника к конкретному планшету.
